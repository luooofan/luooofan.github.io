Shell的启动类型

NOTE：该部分大量引用了Shell 启动类型探究 ── login && interactive这篇文章中的内容。具体内容和细节可以到链接里去看，这里只引一些重要的部分。

什么是Shell

A shell is a program that prints a prompt, reads a line of input from you, and then interprets it as one or more commands to manipulate files or run other programs.

——link

Shell作为一个程序，它也是有一些参数可以在启动时加上的，运行时也支持配置

bash有shopt，zsh有setopt，还可以man zshoptions等等

Shell的类型

由于使用场景的不同，Shell 被分为两个类型：

  • login / non-login：当使用终端登录一台主机时，主机会为我们启动一个 Shell，由于是登录以后启动的，所以是 login Shell。其他情况的 Shell 就是 non-login 的。
  • interactive / non-interactive：日常我们在终端中使用的就是 interactive Shell，它会输出提示符，会有 Job Control 等功能。当我们执行 Shell 脚本时的 Shell 便是 non-interactive 的。

这两个类型影响的是 Shell 的启动文件 (startup files)

Vim执行shell命令

我们知道在Vim命令行模式下可以用:!{cmd}的方式来执行shell命令，也可以用:r !{cmd}等命令来和当前缓冲区交互。

在WSL下如果按这样配置了和Windows系统剪贴板交互的命令，也就是在~/.zshrc定义了两个”别名”alias，如果想在vim中用，却发现用不了，zsh找不到命令。

结合上文，也就是说vim在默认情况下执行shell命令时是启动了non-interactive，non-login的shell在执行，对于zsh来说，这种情况下并不会加载zshrc，不过会加载zshenv，所以只要把那两个别名放到合适的启动文件中即可。

另外，对于bash来说，可以使用shopt expand_aliases来在non-interactive的shell下拓展别名。

当然，指定vim启动一个interactive类型的shell来执行命令-ic也不是不行，会加载zshrc，找得到命令，但是会遇到执行完命令后自动把vim挂起（suspend）的问题，还需要手动fg把vim回复到前台运行。
